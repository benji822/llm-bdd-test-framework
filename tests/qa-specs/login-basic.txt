Feature: Login with credentials

Users authenticate with email and password to access their account.

Happy path:
- User opens the login page.
- User enters valid email and password.
- User clicks the submit button.
- User is redirected to dashboard.
- Dashboard displays welcome message.

Invalid password:
- User opens the login page.
- User enters valid email with incorrect password.
- User clicks the submit button.
- System shows error message "Invalid credentials provided".
- Email field retains the entered value.
- Submit button remains enabled for retry.

Unknown email:
- User opens the login page.
- User enters non-existent email address.
- User enters any password.
- User clicks the submit button.
- System shows error message "Invalid credentials provided".
- Form remains enabled for retry.
